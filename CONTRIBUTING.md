# Contributing to AgentReady

First off, thank you for considering contributing to **AgentReady**! It's people like you that make this toolkit such a great tool.

## Local Setup

### Clone and Install

```bash
# Clone the repository
git clone https://github.com/vb-nattamai/agent-ready.git
cd agent-ready

# Create a virtual environment
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate

# Install dependencies (editable with LLM support)
pip install -e '.[ai,dev]'

# Set up API key for your chosen provider
export ANTHROPIC_API_KEY="sk-ant-..."   # anthropic (default)
export OPENAI_API_KEY="sk-..."           # openai
export GOOGLE_API_KEY="..."              # google
export GROQ_API_KEY="gsk_..."            # groq
export MISTRAL_API_KEY="..."             # mistral
export TOGETHER_API_KEY="..."            # together
# ollama: no key needed — runs locally
```

## Adding a New Language Template

### Step 1: Create the Template

Create a new file in `templates/` following the naming pattern:
- `tool.{language}.template.{ext}` for tool/MCP templates
- Examples: `tool.ruby.template.rb`, `tool.php.template.php`

```bash
# Example: Add Ruby support
touch templates/tool.ruby.template.rb
```

### Step 2: Implement the Template

Follow the MCP (Model Context Protocol) tool pattern:

```ruby
# Generated by AgentReady (https://github.com/vb-nattamai/agent-ready)
# Tool Template: Ruby
#
# This is a scaffold for creating MCP-compatible tools in Ruby.

def {{TOOL_NAME}}({{PARAM_1_NAME}}, {{PARAM_2_NAME}} = {{PARAM_2_DEFAULT}})
  """{{TOOL_DESCRIPTION}}"""
  
  # Input validation
  raise ArgumentError, "{{PARAM_1_NAME}} required" if {{PARAM_1_NAME}}.nil?
  
  begin
    # Tool logic
    result = execute_tool_logic({{PARAM_1_NAME}}, {{PARAM_2_NAME}})
    
    { success: true, data: result, error: nil }
  rescue StandardError => e
    { success: false, data: nil, error: e.message }
  end
end

# Tool metadata (MCP-compatible)
TOOL_METADATA = {
  name: "{{TOOL_NAME}}",
  description: "{{TOOL_DESCRIPTION}}",
  input_schema: {
    type: "object",
    properties: {
      {{PARAM_1_NAME}}: { type: "{{PARAM_1_JSON_TYPE}}", description: "{{PARAM_1_DESCRIPTION}}" }
    },
    required: ["{{PARAM_1_NAME}}"]
  }
}.freeze
```

### Step 3: Register the Language in the Analyser

Edit `src/agent_ready/analyser.py` and update:

1. **`LANGUAGE_EXTENSIONS`** dict to include `.rb` → `"Ruby"`
2. **`FRAMEWORK_INDICATORS`** dict if Ruby frameworks should be detected
3. **`lang_to_template`** mapping to include `"Ruby": "tool.ruby.template.rb"`
4. **`lang_to_output`** mapping to include `"Ruby": "tools/example_tool.rb"`

Example changes:

```python
LANGUAGE_EXTENSIONS: dict[str, list[str]] = {
    # ... existing entries ...
    "Ruby": [".rb"],  # ADD THIS
}

FRAMEWORK_INDICATORS: dict[str, list[str]] = {
    # ... existing entries ...
    "Rails": ["rails", "rails-api"],  # ADD THIS
    "Sinatra": ["sinatra"],  # ADD THIS
}

lang_to_template = {
    # ... existing entries ...
    "Ruby": "tool.ruby.template.rb",  # ADD THIS
}

lang_to_output = {
    # ... existing entries ...
    "Ruby": "tools/example_tool.rb",  # ADD THIS
}
```

### Step 4: Test the New Language

```bash
# Create a minimal Ruby project structure
mkdir -p /tmp/test-ruby/{lib,spec}
echo "source 'https://rubygems.org'; gem 'rails'" > /tmp/test-ruby/Gemfile
echo "puts 'hello'" > /tmp/test-ruby/lib/app.rb

# Run transformer
agent-ready --target /tmp/test-ruby --dry-run

# Verify ruby tool was detected
agent-ready --target /tmp/test-ruby

# Check that tools/example_tool.rb was created
ls -la /tmp/test-ruby/tools/example_tool.rb
```

### Step 5: Update Template (if needed)

If the tool template needs language-specific adjustments based on detected framework:

1. Add framework detection logic to `_detect_frameworks()` in the transformer
2. Pass framework info to template substitution
3. Customize output in the generated tool file

## Adding a New Platform-Specific Agent Format

### Step 1: Create Agent Definition

Create `.{platform}/agents/agent-ready.agent.md`:

```bash
# Example: For Cursor IDE
mkdir -p .cursor/agents
touch .cursor/agents/agent-ready.agent.md
```

### Step 2: Add Agent Instructions

Model after existing `.github/agents/agent-ready.agent.md` or `.claude/agents/agent-ready.agent.md`:

```markdown
# AgentReady — Transformer Agent

> **Platform:** Cursor IDE
> **Version:** 1.1.1
> **Purpose:** Transform any repository into an AI-agent-ready codebase

## Identity

You are the **AgentReady Transformer** for Cursor IDE.

## Instructions

[Your platform-specific instructions here]
```

### Step 3: Create Template File

Create a new template in `templates/`:

```bash
touch templates/CURSOR.template.md
```

Follow the pattern of `AGENTS.template.md` or `CLAUDE.template.md`.

### Step 4: Update the Generator

In `src/agent_ready/generator.py`, update the `generate_all()` method (or equivalent):

```python
def generate_all(self) -> list[str]:
    """Generate all agent-ready files."""
    self.generate_context_json()
    self.generate_agents_md()
    self.generate_claude_md()
    self.generate_cursor_md()  # ADD THIS
    self.generate_system_prompt()
    self.generate_mcp_json()
    self.generate_tool_templates()
    return self.generated

def generate_cursor_md(self) -> None:
    """Generate CURSOR.md for Cursor IDE."""
    # Similar to generate_claude_md()
    template_path = TEMPLATES_DIR / "CURSOR.template.md"
    if template_path.exists():
        content = template_path.read_text(encoding="utf-8")
        filled = self._fill_template(content)
        self._write_file("CURSOR.md", filled)
```

## Pull Request Checklist

Before submitting a PR, ensure:

- [ ] Templates use `{{PLACEHOLDER}}` syntax consistently
- [ ] New language templates follow the dual-format TOOL_SCHEMA pattern
  - `success` (boolean), `data` (result), `error` (string | null)
- [ ] `analyser.py` detects the new language correctly
  - Run transformer on test repo with new language
  - Verify correct detection in output
- [ ] README.md updated if new usage mode added
- [ ] No hardcoded paths or usernames in code
- [ ] Templates don't reference files/paths that don't exist
- [ ] All 22 tests passing (`pytest`)
- [ ] Code linted with `ruff` (`ruff check src/ tests/ && ruff format --check src/ tests/`)

## How Can I Contribute?

### Reporting Bugs

Before creating a bug report, check the issue list. When creating one, include:

* Clear and descriptive title
* Exact steps to reproduce
* Specific examples
* What you observed vs. what you expected
* Your environment (OS, Python version, etc.)

### Suggesting Enhancements

Create an issue with:

* Clear and descriptive title
* Step-by-step description
* Specific examples
* Why this would be useful

### Adding Examples

Add new examples in `docs/EXAMPLES.md`:

```markdown
## Example: {Language + Framework}

**Project:** [Project Name](link)

```bash
# Commands to run transformer
agent-ready --target /path/to/project
```

**Generated files:**
- ✅ AGENTS.md
- ✅ CLAUDE.md
- ... etc
```

## Commit Message Guidelines

Release automation in `.github/workflows/release.yml` parses Conventional Commit prefixes (`feat:`, `fix:`, and `BREAKING CHANGE:`). Commitlint is not currently enforced by CI in this repository.

* Use present tense ("Add feature" not "Added feature")
* Use imperative mood ("Move cursor to..." not "Moves cursor to...")
* Limit first line to 72 characters
* Reference issues/PRs liberally after first line

Example:
```
Add Ruby/Rails support to transformer

- Detect .rb files and Gemfile  
- Generate Rails-specific agent instructions
- Add tool.ruby.template.rb scaffold

Fixes #42
```

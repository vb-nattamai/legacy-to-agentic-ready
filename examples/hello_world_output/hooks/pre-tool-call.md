---
name: pre-tool-call
trigger: Before any tool call that writes files
---

## Purpose

Guards file-write operations in this Flask/Python repo by validating target paths against configured restrictions and loading current session state before any file is modified.

## Actions

1. Load `agent-context.json` to retrieve the current session state (active branch, in-progress edits, and any pending operations) and cross-reference the target write path against `restricted_write_paths` from the repo configuration — currently empty, but validate that writes to `app.py` do not unintentionally rename the Flask app object or alter the top-level `GREETING` assignment without a corresponding note, as these are known pitfalls.
2. Check `memory/schema.md` for the session state contract to confirm the write operation is consistent with the expected file structure, particularly ensuring that any new or modified file under `tests/` follows the import convention for the single-file `app` module (i.e., `from app import app`) and that `requirements.txt` or `requirements-dev.txt` edits include `pytest` and `pytest-cov` if either file is the write target.

## Context loaded

- **agent-context.json**: Current agent state including active task, branch context, and any previously flagged write conflicts.
- **memory/schema.md**: Session state contract defining expected structure for files in this repo, import conventions for the `app` module, and constraints on environment-variable-dependent code such as `GREETING`.

## Skipped when

- `AGENT_SKIP_HOOKS=true` environment variable is set.
- The tool call is a read-only operation (e.g., `read_file`, `list_directory`, `search`) and does not write or truncate any file.
- The target file is a temporary file (e.g., matching `*.tmp`, `*.pyc`, or paths under `__pycache__/`) generated by the Python runtime or pytest.
